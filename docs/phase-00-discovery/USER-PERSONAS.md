## 1. Artist Alex

### Profile

| Attribute | Value |
|-----------|-------|
| **Name** | Alex Müller |
| **Age** | 28 |
| **Location** | Frankfurt, Hessen |
| **Profession** | Professional DJ / Electronic Music Producer |
| **Income** | €35,000/year from music (variable) |
| **Tech Comfort** | High - uses multiple apps daily |

### Background

Alex has been DJing professionally for 6 years, playing a mix of techno and house music at clubs, private events, and corporate parties in the Rhein-Main area. He supplements gig income with production work and occasional music lessons. His reputation is built largely through Instagram and word-of-mouth referrals.

### Goals

1. **More consistent bookings** - Fill calendar gaps, especially weeknights
2. **Better paying gigs** - Access to corporate events and upscale venues
3. **Reduced admin time** - Spend less time on emails, more on music
4. **Grow reputation** - Build verifiable track record with reviews
5. **Direct venue relationships** - Cut out middleman booking agents

### Pain Points

- **Platform fatigue:** Managing profiles on Eventpeppers (€150/year), Gigmit (€228/year), plus social media
- **Slow responses:** Venues often don't respond to inquiries
- **Payment chasing:** Following up on invoices is tedious
- **Last-minute cancellations:** No accountability for no-shows
- **Visibility:** Hard to reach organizers outside existing network

### Behaviors

- Checks phone constantly for booking inquiries
- Posts performance clips to Instagram stories
- Maintains spreadsheet for tracking bookings and payments
- Prefers WhatsApp for quick communication
- Searches "DJ gigs Frankfurt" monthly hoping to find opportunities

### Quote

> "I'm spending more time managing my booking platforms than actually making music. I need one place that just works."

### Feature Priorities

| Priority | Feature |
|----------|---------|
| P0 | Professional profile with audio/video samples |
| P0 | Availability calendar |
| P0 | Booking request notifications |
| P1 | Rating/review system to build reputation |
| P1 | Analytics to see profile views |
| P2 | Contract templates |
| P3 | In-platform payment tracking |

## 2. Venue Viktor

### Profile

| Attribute | Value |
|-----------|-------|
| **Name** | Viktor Schneider |
| **Age** | 42 |
| **Location** | Wiesbaden, Hessen |
| **Role** | Owner/Manager of mid-size club |
| **Venue Type** | 300-capacity club with bar and stage |
| **Tech Comfort** | Medium - uses basics but not early adopter |

### Background

Viktor has run "Club Nachtwerk" for 8 years. The venue hosts live music on weekends and DJ nights during the week. He personally handles entertainment booking, which takes 10+ hours per week. The club's success depends on quality programming that draws crowds.

### Goals

1. **Quality entertainment** - Find reliable, talented artists
2. **Fill calendar** - No empty nights, especially midweek
3. **Reduce booking time** - Streamline the search and negotiation process
4. **Build artist roster** - Develop relationships with proven performers
5. **Promote events** - Get bookings visible to potential attendees

### Pain Points

- **Finding artists is hard:** Instagram searching, asking for recommendations, checking Eventpeppers one by one
- **Quality uncertainty:** Can't verify if artist is actually good before booking
- **Communication scattered:** Negotiations happen via email, WhatsApp, Instagram DMs
- **No-shows:** Artists cancel last minute with no consequences
- **Admin overhead:** Contracts, payments, scheduling - all manual

### Behaviors

- Scrolls through Instagram for artist discovery
- Asks other venue owners for recommendations
- Maintains paper calendar for bookings
- Prefers phone calls over email
- Posts event flyers to Facebook

### Quote

> "I need good acts for Friday night. I don't have time to search through 500 Instagram profiles hoping someone is available."

### Feature Priorities

| Priority | Feature |
|----------|---------|
| P0 | Search artists by genre, location, availability |
| P0 | Artist profiles with samples and reviews |
| P0 | Send booking requests quickly |
| P1 | Venue calendar showing booked/available |
| P1 | Ratings from other venues |
| P2 | Quick re-book for past artists |
| P3 | Payment processing |

## 3. Organizer Olivia

### Profile

| Attribute | Value |
|-----------|-------|
| **Name** | Olivia Weber |
| **Age** | 35 |
| **Location** | Mainz, Rheinland-Pfalz |
| **Role** | Event Planner / Wedding Coordinator |
| **Clients** | 25-30 events per year |
| **Tech Comfort** | High - uses project management tools |

### Background

Olivia runs a boutique event planning business, specializing in weddings and corporate events. Each event requires coordinating multiple vendors: venue, catering, entertainment, photography, etc. She prides herself on quality and personal service.

### Goals

1. **Efficient sourcing** - Find artists AND venues in one place
2. **Availability alignment** - See who's free on client's date
3. **Compare options** - Present clients with choices and pricing
4. **Manage communications** - Track all conversations per event
5. **Budget management** - Keep costs organized per event

### Pain Points

- **Multiple platforms:** Uses Eventinc for venues, Eventpeppers for artists, Hochzeitslocations.de for weddings
- **Availability puzzle:** Calls multiple vendors to check date availability
- **Price opacity:** Hard to compare pricing across vendors
- **Client communication:** Forwarding quotes and options is tedious
- **Contract management:** Different contracts from every vendor

### Behaviors

- Uses Notion for project management
- Creates comparison spreadsheets for clients
- Attends wedding fairs to network with vendors
- Active on LinkedIn for corporate client acquisition
- Prefers email for documentation purposes

### Quote

> "My clients trust me to find the perfect combination of venue and entertainment. I can't do that efficiently when I'm juggling five different websites."

### Feature Priorities

| Priority | Feature |
|----------|---------|
| P0 | Search both artists and venues |
| P0 | Filter by date availability |
| P0 | Save favorites to shortlists |
| P0 | Unified messaging per booking |
| P1 | Price comparison view |
| P1 | Event-based organization |
| P2 | Client sharing (shareable shortlists) |
| P3 | Budget tracking per event |

| **Age** | 26 |
| **Location** | Frankfurt, Hessen |
| **Role** | Marketing professional |
| **Interests** | Live music, nightlife, cultural events |
| **Tech Comfort** | Very high - digital native |

2. **Follow artists** - Get notified when favorites play locally

- Uses Google Maps to find nearby venues
- Asks friends for recommendations via WhatsApp
- Rarely plans more than 1 week ahead

### Quote

## Key Insights

1. **Artists and Venues have complementary needs** - Artist wants gigs, venue needs entertainment
2. **Organizers need unified platform** - Currently using 3+ platforms
3. **Customers are secondary** - Focus on B2B marketplace first
4. **Quality signals matter** - Reviews and verification reduce friction
5. **Admin reduction** - All personas want less manual work
